斜杠命令

学习使用 Claude Code 内置的斜杠命令进行上下文管理、会话控制和配置。

入门 15 分钟

斜杠命令是在交互式会话中控制 Claude Code 行为最快的方式。在任意提示符下输入 / 即可查看完整列表，或输入几个字母进行筛选。本模块涵盖了你每天都会使用的内置命令。

发现命令

在提示符下输入 /，菜单就会显示所有可用命令。开始输入以筛选 — /co 缩小到 /compact、/color、/config、/context、/cost、/copy。方向键导航，回车选择。当前设置下不可用的命令会自动隐藏，所以你只会看到能用的。

刚接触 Claude Code？试试 /powerup — 它会运行带有动画演示的交互式课程，直接在 CLI 中带你了解关键功能。

有些命令直接接受参数：/compact focus on the API layer、/model opus、/effort high、/rename auth-refactor。其他如 /context、/cost 和 /status 则无需参数立即运行。

/compact focus on the payment service

命令分类

内置命令分为几个类别。了解类别有助于你找到合适的命令，而无需记住所有命令。

上下文管理 控制 Claude 可以看到多少对话内容。

  • /context — 显示上下文使用情况的可视化网格
  • /compact — 压缩对话。传入指令控制保留什么：/compact keep the migration plan, drop the debugging
  • /clear — 完全重新开始

会话工具 让你管理和回顾工作。

  • /rename my-feature — 给会话起一个可读的名称
  • /resume — 恢复之前的会话
  • /branch — 创建并行对话，探索替代方案而不丢失当前状态
  • /rewind — 回滚到更早的时间点。/undo 是 /rewind 的别名。
  • /export — 将会话保存到文件或剪贴板
  • /recap — 生成会话中发生内容的一行摘要。当你离开后返回终端时也会自动运行。

配置 命令在会话中调整 Claude 的行为。

  • /model — 切换可用模型，如 Sonnet、Opus、Haiku 以及其他别名如 best 或 opusplan
  • /effort — 设置推理深度：low、medium、high、xhigh、max（仅会话）或 auto
  • /permissions — 管理 Claude 无需询问即可执行的操作
  • /config — 打开设置菜单
  • /theme（v2.1.118 新增）— 创建和切换命名自定义主题。主题以 JSON 文件存储在 ~/.claude/themes/ 中，也可以直接手动编辑。
  • /tui — 在对话中切换经典和无闪烁全屏渲染。也可作为 tui 设置使用。
  • /focus — 切换专注模式以实现无干扰输入。
  • /less-permission-prompts — 扫描最近的对话记录，查找常见的只读 Bash 和 MCP 工具调用，然后建议将允许列表添加到 .claude/settings.json 中，以减少未来的权限提示。

诊断 在出问题时提供帮助。

  • /cost — 显示会话费用、时长、代码更改和 token 使用量
  • /usage-credits（v2.1.144 新增）— 查看和管理账户上的使用额度。从 /extra-usage 重命名而来；旧名称仍然有效。
  • /status — 显示版本、模型和账户信息
  • /doctor — 检查安装健康状况
  • /diff — 打开未提交更改的交互式查看器，在提交前查看 Claude 做了什么

发布前审查 在代码离开分支前检查你的工作。

  • /code-review（v2.1.147 新增）— 审查当前差异的正确性错误，并报告发现而不编辑文件。接受努力级别（/code-review high 返回更广泛的覆盖；/code-review low 返回更少但置信度更高的发现）和 --comment 以将发现作为内联评论发布到当前的 GitHub PR。传入路径或 PR 引用以审查特定目标。这是一个内置技能；/code-review --fix 将发现应用到你的工作树。从 v2.1.154 开始，/simplify 是一个独立的仅清理技能，应用修复而不寻找错误 — 使用 /code-review 来查找错误。
  • /code-review ultra — 在云端沙箱中使用并行分析和评审运行深度多代理代码审查。无参数时审查当前分支；传入 PR 编号（/code-review ultra 123）来获取和审查 GitHub PR。/ultrareview 是别名。Pro 和 Max 包含少量免费运行，之后消耗使用额度。
  • /review — 对更改集进行只读深入检查
  • /security-review — 对待定更改进行安全重点审查

专注与检查工作流

markdown
/context Read `README.md`, `CLAUDE.md`, and `package.json`, then summarize the project conventions before making changes.

任务间上下文重置

完成任务后压缩上下文，以便以干净的状态开始下一个任务。

markdown
/compact summarize what was completed and preserve any open questions, then drop implementation details

发布前正确性扫描

在当前差异上运行内置的 /code-review 技能，然后将发现发布到打开的 PR。自 v2.1.147 可用 — 从 /simplify 重命名而来。从 v2.1.154 起 /simplify 是一个独立的仅清理技能。

markdown
/code-review high # Once the findings look reasonable, post them to the PR: /code-review high --comment
