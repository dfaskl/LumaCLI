斜杠命令

学习使用 LumaCLI 内置的斜杠命令，高效管理上下文、控制会话行为与调整配置。

入门级 15 分钟

斜杠命令是在交互式会话中控制 LumaCLI 行为的最快方式。在任意提示符处输入 / 即可查看完整列表，或输入几个字母进行过滤。本模块涵盖了你每天都会用到的内置命令。

发现命令

在提示符处输入 /，会弹出一个包含所有可用命令的菜单。继续输入即可过滤 —— 例如 /co 会筛选出 /compact、/color、/config、/context、/cost、/copy。使用方向键导航，回车确认。对于当前环境不适用的命令会自动隐藏，只显示可用的选项。

如果你是 LumaCLI 的新手，可以试试 /powerup —— 它会运行交互式教程，通过动画演示带你快速掌握核心功能。

部分命令可以直接接收参数：/compact focus on the API layer、/model opus、/effort high、/rename auth-refactor。而 /context、/cost 和 /status 这类命令则无需参数，直接运行即可。 

/compact focus on the payment service

命令分类

内置命令按功能划分为几个类别。了解这些分类，有助于你在不刻意记忆所有命令的情况下，快速找到所需功能。

上下文管理 控制 LumaCLI 在对话中可查看的内容范围。

  • /context — 以可视化网格展示当前上下文使用情况
  • /compact — 压缩对话历史。可传入指令控制保留内容：/compact keep the migration plan, drop the debugging
  • /clear — 完全清空上下文，重新开始

会话工具 让你管理和回顾工作。

  • /rename my-feature — 为当前会话命名
  • /resume — 恢复之前的会话
  • /branch — 创建并行对话，在不丢失当前状态的前提下探索替代方案
  • /rewind — 回滚到更早的节点。/undo 是它的别名。
  • /export — 将会话导出为文件或复制到剪贴板
  • /recap — 生成会话的单行摘要。当你离开终端一段时间后返回时，也会自动运行。

配置 在会话中调整 LumaCLI 的行为。

  • /model — 在可用模型间切换，如 Sonnet、Opus、Haiku，或别名如 best、opusplan
  • /effort — 设置推理深度：low、medium、high、xhigh、max（仅当前会话）或 auto
  • /permissions — 管理 LumaCLI 无需询问即可执行的操作
  • /config — 打开设置菜单
  • /theme（v2.1.118 新增）— 创建并切换自定义主题。主题以 JSON 文件形式存储在 ~/.pai/themes/ 下，也可直接手动编辑。
  • /tui — 在经典模式和无闪烁全屏渲染之间切换
  • /focus — 切换专注视图，实现无干扰输入
  • /less-permission-prompts — 扫描最近的会话记录，针对常见的只读 Bash 和 MCP 工具调用提出允许列表建议，减少后续权限询问

诊断 当遇到问题时提供排查帮助。

  • /cost — 显示会话成本、耗时、代码变更和令牌使用情况
  • /usage-credits（v2.1.144 新增）— 查看和管理账户使用额度。原名 /extra-usage，旧名称仍可用。
  • /status — 显示版本、模型和账户信息
  • /doctor — 检查安装健康状况
  • /diff — 打开交互式查看器查看未提交的更改，提交前预览 LumaCLI 的改动很有用

发布前审查 在代码离开你的分支之前检查工作质量。

  • /code-review（v2.1.147 新增）— 审查当前差异的正确性缺陷，报告发现而不修改文件。接受强度级别（/code-review high 返回更广泛的覆盖范围；/code-review low 返回更少但更高置信度的发现），以及 --comment 将发现作为内联评论发布在当前 GitHub PR 上。这是内置技能；/code-review --fix 会将发现应用到工作树。从 v2.1.154 起，/simplify 是一个独立的仅修复技能，用于应用修复而不排查缺陷 —— 查找缺陷请使用 /code-review。
  • /code-review ultra — 在云沙箱中运行深度、多智能体代码审查，使用并行分析和批判。无参数时审查当前分支；传入 PR 编号（如 /code-review ultra 123）可获取并审查 GitHub PR。/ultrareview 是别名。Pro 和 Max 版本包含少量免费运行次数，之后将消耗使用额度。
  • /review — 对变更集进行只读的深度检查
  • /security-review — 对待处理的更改进行安全重点审查

/code-review high
